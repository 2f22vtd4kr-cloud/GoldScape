# Terrain Map Transparency

## The Rule
All terrain map PNGs generated by AI image generators come with solid white backgrounds. On a dark-themed page, this shows as an ugly white square behind the 3D model. **Always process new terrain maps before committing.**

**Why:** The image generator renders the 3D terrain model on a white canvas. The PNG has no alpha channel. Corner-floodfill with ImageMagick removes the white without touching interior near-white pixels (mountain tops, buildings).

## How to Apply
For every new terrain map PNG, run:
```bash
magick input.png \
  -fuzz 5% \
  -draw "color 0,0 floodfill" \
  -draw "color 0,1023 floodfill" \
  -draw "color 1023,0 floodfill" \
  -draw "color 1023,1023 floodfill" \
  output-nobg.png
```
Save as `terrain-map-<country>-nobg.png` alongside the original in `attached_assets/generated_images/`. The import in `countries.ts` must point to the `-nobg.png` file.

## CSS Consequences
After making PNGs transparent:
- **Remove** `box-shadow` and `border` from `.iso-tile` in `index.css` — those apply to the rectangular bounding box, not the terrain shape, and look wrong with transparent corners.
- **Keep** `filter: drop-shadow()` on the `<img>` tag — it respects alpha and casts the shadow along the terrain silhouette.
- The `.iso-tile` class should only keep `transform: rotateX(52deg) rotateZ(-18deg)` and `transform-style: preserve-3d`.

## CountryPage Section Overflow
The hero section in `CountryPage.tsx` uses `overflow-x-hidden` (not `overflow-hidden`) so the 3D-transformed terrain can visually extend slightly past its layout box without being clipped on the vertical axis or triggering horizontal scroll.
